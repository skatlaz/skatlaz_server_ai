Boas práticas

O servidor deve ser seguro, rápido e observável.

Segurança

Use CSRF quando possível, autenticação por token para API, rate limit e logs por usuário.

Performance

Use workers para WebDiver, indexação, embeddings e fine-tuning.

Observabilidade

Registre payload, agente, coleção, tempo de execução, erro e custo aproximado por chamada.

Regra de ouro

Skatlaz Server deve responder rápido no chat e mover processos longos para filas. O Client Ask deve ser leve, bonito e direto.

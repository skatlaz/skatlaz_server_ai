Buenas prácticas

El servidor debe ser seguro, rápido y observable.

Seguridad

Use CSRF cuando sea posible, autenticación por token para API, rate limit y logs por usuario.

Performance

Use workers para WebDiver, indexación, embeddings y fine-tuning.

Observabilidad

Registre payload, agente, colección, tiempo de ejecución, error y costo aproximado por llamada.

Regla de oro

Skatlaz Server deve responder rápido no chat e mover processos longos para filas. O Client Ask deve ser leve, bonito e direto.

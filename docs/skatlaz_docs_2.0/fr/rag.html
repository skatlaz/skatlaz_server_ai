Construire le RAG

Le RAG transforme fichiers et pages en connaissance recherchable. Skatlaz Server doit utiliser DocumentSource, RagCollection et RagChunk comme base.

1Source
2Extract
3Chunk
4Embedding
5Search
python manage.py skatlaz_index_rag
POST /ai/rag/index-source/ {"source_id": 1}
GET /ai/rag/search/?q=texto&collection_id=1

Bonnes pratiques RAG

Chunks

Utilisez des chunks entre 500 et 1200 tokens.

Métadonnées

Stockez titre, URL, langue, date et collection.

Prompt

N’envoyez jamais tout le document au LLM ; récupérez seulement les passages pertinents.

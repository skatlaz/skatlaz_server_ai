Bonnes pratiques

Le serveur doit être sécurisé, rapide et observable.

Sécurité

Utilisez CSRF si possible, authentification par token pour API, rate limit et logs par utilisateur.

Performance

Utilisez des workers pour WebDiver, indexation, embeddings et fine-tuning.

Observabilité

Journalisez payload, agent, collection, temps d’exécution, erreur et coût approximatif par appel.

Règle d’or

Skatlaz Server deve responder rápido no chat e mover processos longos para filas. O Client Ask deve ser leve, bonito e direto.

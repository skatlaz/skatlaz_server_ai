Construindo o RAG

O RAG transforma arquivos e páginas em conhecimento pesquisável. O Skatlaz Server deve manter DocumentSource, RagCollection e RagChunk como base.

1Source
2Extract
3Chunk
4Embedding
5Search
python manage.py skatlaz_index_rag
POST /ai/rag/index-source/ {"source_id": 1}
GET /ai/rag/search/?q=texto&collection_id=1

Boas práticas de RAG

Chunks

Use chunks entre 500 e 1200 tokens.

Metadados

Guarde título, URL, idioma, data e coleção.

Prompt

Nunca envie o documento inteiro ao LLM; recupere apenas trechos relevantes.
